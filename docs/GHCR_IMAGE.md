# GHCR Image

CrayZ publishes the server image to:

```text
ghcr.io/noscopestudios/crayz-server
```

The Docker workflow builds `./Dockerfile` from the repository root and publishes on pushes to `main` or `master`, semantic version tags such as `v0.1.0`, and manual runs.

`latest` is published only from the default branch. Branch, tag, semantic version, and short SHA tags are generated by `docker/metadata-action`.

Pull example:

```bash
docker pull ghcr.io/noscopestudios/crayz-server:latest
```
